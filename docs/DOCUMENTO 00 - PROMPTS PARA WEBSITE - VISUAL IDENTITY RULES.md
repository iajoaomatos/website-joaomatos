# **PROMPTS PARA WEBSITE**

\# VISUAL IDENTITY RULES

Follow all previous project documents.

\------------------------------------------------------------

GOAL

The purpose of this document is NOT to redesign João's personal brand.

The visual identity already exists.

This project should feel like a natural extension of:

\- joaomatosdigital.pt  
\- JoãoMatos\_VisualCV

\------------------------------------------------------------

VISUAL PRIORITIES

Always prioritise:

PRIORITY 01

JoaoMatosDigital.pt

\+

JoaoMatos\_VisualCV

(Visual Identity)

\------------------------------------------------------------

PRIORITY 02

Provided Design System

(Layouts, Components and UX patterns)

\------------------------------------------------------------

IMPORTANT

If any conflict exists between:

\- JoaoMatosDigital.pt  
\- JoaoMatos\_VisualCV

and

\- the provided Design System

Always prioritise:

JoaoMatosDigital.pt

\+

JoaoMatos\_VisualCV.

\------------------------------------------------------------

VISUAL PRINCIPLES

The final implementation should feel:

\- minimal  
\- elegant  
\- premium  
\- modern  
\- technological  
\- professional  
\- human

Avoid:

\- trendy designs  
\- information overload

\------------------------------------------------------------

TYPOGRAPHY

Always prioritise consistency with:

\- joaomatosdigital.pt  
\- JoãoMatos\_VisualCV

Never introduce radically different font combinations.

Typography should feel like a natural extension of the existing personal brand.

\------------------------------------------------------------

COLOUR PALETTE

Always prioritise the existing colour palette.

Avoid:

\- new primary colours  
\- excessive gradients  
\- highly saturated colours  
\- unnecessary accent colours

If accent colours are required, they should naturally complement the existing visual identity.

\------------------------------------------------------------

DESIGN SYSTEM

The Design System should only be used as a reference for:

\- layouts  
\- components  
\- spacing  
\- responsiveness  
\- animations  
\- accessibility  
\- interaction patterns

The Design System should NEVER redefine João's personal brand.

\------------------------------------------------------------

ANIMATIONS

Prefer:

\- subtle transitions  
\- progressive reveals  
\- elegant hover effects

Avoid:

\- large animations  
\- distracting interactions  
\- excessive motion

Animations should support understanding and never become the centre of attention.

\------------------------------------------------------------

WHITESPACE

Always prefer:

more whitespace.

Less content is preferable to crowded layouts.

\------------------------------------------------------------

THE FEELING OF THE WEBSITE

The final implementation should feel:

Simple, but modern

Elegant, but technological

Human.

Visitors should immediately feel that they are navigating through different parts of the same personal brand ecosystem.

The objective is NOT visual similarity.

The objective is visual consistency.

\------------------------------------------------------------

MAIN OBJECTIVE

Visitors should immediately understand that this website belongs to:

\- joaomatosdigital.pt  
\- JoãoMatos\_VisualCV

without explicitly being told so.

\------------------------------------------------------------

FINAL PRINCIPLE

Consistency is always preferable to originality.

Never sacrifice consistency for visual experimentation.
